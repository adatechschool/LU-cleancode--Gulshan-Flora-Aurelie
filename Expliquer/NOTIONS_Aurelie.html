CLEAN CODE

PREMIERES REGLES

  • Clean code is expressive (noms de variables clairs, indentation...)
  • Clean code is simple --KISS--
  • Not repeat yourself : si cela va reservir, fais-en une fonction

EN RESUME : "THE ONLY VALID MEASUREMENT OF CODE QUALITE : WTFs/MINUT !"

EXPLIQUER

FACTORISATION

Refactorisation, réusinage de code, refactoring ... =
"Changing the structure of existing code without modifying it's behavior" / Répondre aux principes de clean code.

DETTE TECHNIQUE

Métaphore de la dette financière, la dette technique reprend les coûts induits par un développement non optimal, dûs en partie à un dirty code.

ISOFONCTIONNEL

Résultat souhaité sur un code cleanné : le comportement reste le même, seul le code source est modifié (si besoin)

CODE SMELL

Bugs dûs à un dirty code

DESIGN PATTERN, ANTI PATTERN

CODE LEGACY

DRY, KISS, SOLID

  • KISS : Keep It Simple Stupid
  • DRY : Don't Repeat Yourself
  • (YAGNI : You Aren't Gonna Need It)
  • SOLID :
  • Single responsibility principle : each class or function should only have one responsibility
  • Open/ Closed principle : a class should be open for extension, but closed for modification
  • Liskov substitution principle : une classe fille doit être simplement un cas spécial d'utilisation de la classe mère
    (ex : un carré est un rectangle avec une longueur égale à sa hauteur)
